Path A — Vercel CLI (recommended for Vercel)

Step 1 — Install Node.js (5 minutes, one-time)

Vercel's CLI runs on Node.js. This needs to be installed once on your machine.

1 Open https://nodejs.org in your browser
2 Click the green "LTS" button (currently Node 22)
3 Run the installer — accept all defaults
4 Close and reopen PowerShell after installation completes

Verify it worked by running these in PowerShell:

node --version
npm --version

Both should print version numbers. If either says "not recognized", restart your computer and try again.

Step 2 — Clean up local-only files (optional)

Two files (serve.py and upload.html) were created for local development only — for uploading the doctor's photos via the browser. They aren't needed online. Remove them before deploy:

cd C:\Users\hp\Desktop\dr-zeeshan-website
Remove-Item serve.py
Remove-Item upload.html

If you skip this, Vercel will simply serve them as harmless static files. They won't break anything.

Step 3 — Install Vercel CLI (1 minute, one-time)

npm install -g vercel

Step 4 — Deploy

cd C:\Users\hp\Desktop\dr-zeeshan-website
vercel

Vercel will ask 5 quick questions in the terminal:

PromptAnswer
Set up and deploy "…dr-zeeshan-website"?Y
Which scope do you want to deploy to?(Select your username — only option visible)
Link to existing project?N
Project name?dr-zeeshan-raza  (this becomes the URL)
In which directory is your code located?Just press Enter (default ./)
First-time only: Vercel opens your browser to log in. Sign up with email or GitHub — it's free and takes 30 seconds. Return to PowerShell after.

When deployment finishes, you get a preview URL like:

https://dr-zeeshan-raza-abc123.vercel.app

Step 5 — Promote to production URL

vercel --prod

You now get the clean public URL:

https://dr-zeeshan-raza.vercel.app

This is the link you share with the doctor. It's permanent, free, and works on any device.

Step 6 — Updates in the future

Whenever you make changes (new photo, edited text, new section), run this from the project folder:

vercel --prod

The new version goes live in about 30 seconds.

Path B — Vercel via GitHub (no command line)

If you prefer the Vercel website over the command line, this path uses GitHub as the bridge. The advantage is auto-deploy: every code change you push triggers a new live deployment automatically.

1 Sign up at github.com (free)
2 Click "New repository" → name it dr-zeeshan-raza → set to Public → Create
3 On the repo page, click the link "uploading an existing file" → drag the entire contents of the dr-zeeshan-website folder onto the page → scroll down → Commit changes
4 Sign up at vercel.com (free) — sign in with GitHub for one-click linking
5 Click "Add New Project" → select your dr-zeeshan-raza repo from the list → Deploy
6 Wait ~60 seconds → you get the same kind of *.vercel.app URL
Bonus: Every future change you commit to GitHub auto-deploys to Vercel within seconds. Great for collaboration.

Path C — Netlify Drop (30 seconds, no install)

By far the fastest way to put this site online. No account needed for the initial deploy.

1 Open https://app.netlify.com/drop in your browser
2 In File Explorer, navigate to C:\Users\hp\Desktop\
3 Drag the entire dr-zeeshan-website folder onto the Netlify Drop page
4 Wait 15–20 seconds while it uploads and builds
5 Done — you get a URL like https://random-name-12345.netlify.app
This is the link you can immediately share with the doctor. If you'd like to rename the URL (e.g., drzeeshanraza.netlify.app) or be able to update it later, sign up to Netlify after — it'll let you claim the deploy you just dropped.

Custom Domain (Optional, Later)

Once your site is live on Vercel or Netlify, you can attach a real domain like drzeeshanraza.com:

  1. Buy a domain from a registrar (Namecheap or GoDaddy, ~Rs. 3,000/year for .com)
  2. In Vercel/Netlify dashboard → Domains → Add domain → enter your domain
  3. Follow the DNS instructions on screen (usually copy 2 records into your registrar)
  4. Wait 5–60 minutes — your site is live on the custom domain with free HTTPS

Troubleshooting

"node is not recognized" after installing Node.js

Restart your computer. Windows needs to refresh the PATH variable.

Vercel command asks for a different framework

If asked, choose "Other" — this site has no framework, it's plain HTML/CSS/JS.

The intro video doesn't autoplay on the live site

That's a browser policy, not a deployment issue. The preview will autoplay muted; sound only plays after a click. This is the same on every modern browser.

Map iframe shows "refused to connect" locally but not on the live site

Some browsers block iframes on file:// or http://. Live deployments are HTTPS, so the map works fine there — no action needed.

I want to undo a deploy

Vercel: Dashboard → project → Deployments → click any older deployment → Promote to Production. Instant rollback.
Netlify: Same process — Deploys tab → Publish on any older deployment.
